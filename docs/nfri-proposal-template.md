## 2. 연구 목적 및 필요성

### 2.1 연구 목적

드론·UGV 등 이종(異種) 소방 로봇이 단일 오케스트레이터 하에 자율 임무를 수행하는
**개방형 소프트웨어 플랫폼**을 개발한다. 플랫폼은 오픈소스(Apache 2.0)로 공개하여
국내 소방 로봇 산업 생태계의 공통 기반 기술로 자리잡는 것을 목표로 한다.

### 2.2 연구 배경

**현황과 문제점**:

소방청은 2026년 R&D 예산 503억 원(전년 대비 +64.9%)을 편성하고 무인소방로봇 100대
보급 계획(2027~2030)을 추진 중이다. 그러나 현재 소방 로봇은 다음 한계를 가진다:

1. **단일 플랫폼 의존**: HR-셰르파(현대로템), BOLA(LIG넥스원) 등 제조사별 독자 시스템으로
   상호운용 불가. 여러 종류의 로봇을 동시에 투입해도 통합 지휘 불가
2. **수동 제어 중심**: 현장 소방관이 각 로봇을 개별 조작 → 지휘 부담 과중
3. **오케스트레이션 공백**: 드론(항공)+UGV(지상) 이종 협업 시 임무 배분·충돌 회피·
   상황 공유를 자동화하는 소프트웨어 표준 부재

**정책 근거**:

- 소방청 10대 전략과제 5번: "육상·수상·항공 무인화 장비 자율 운용 기술 개발"
- 소방청 10대 전략과제 4번: "AI·디지털 기반 현장지휘체계 구축"
- 무인소방로봇 100대 보급 시 로봇 간 분산 오케스트레이션 수요 직접 발생

## 3. ARGOS 기술 현황 (선행 연구)

### 3.1 개발 이력

| 단계 | 기간 | 주요 성과 |
|------|------|----------|
| Phase A~C (v1.0~3.0) | 2026.02~03 | ROS2 기반 UGV 자율탐색, 멀티로봇 TF 격리, Zenoh 통신 |
| Phase D (v3.5) | 2026.03~ | PX4 드론 통합, CBBA 임무 할당기(22 tests), Platform 추상화(53 tests) |
| Phase E (v4.0) | 예정 | MARL 학습, CBBA 고도화(번들), Isaac Lab |
| Phase F (v5.0) | 예정 | TurtleBot4 실물, 달성군 야외 실증 |

### 3.2 4계층 아키텍처

```
[Orchestrator Layer]
  └─ orchestrator_node.py: 임무 할당, 상황 종합, CBBA 연동

[Mission Layer]
  └─ 정찰 / 진압보조 / 구조지원 / 위험물 (교체식 BT 모듈)

[Core Services Layer]
  └─ Nav2 경로계획 + slam_toolbox SLAM + Zenoh 군집통신 + 상태관리

[Platform Layer]  ← 인터페이스 통일, 구현 분리
  ├─ UGVPlatform (TurtleBot4 / HR-셰르파 대응)
  └─ PX4Platform (드론 TrajectorySetpoint)
```

## 4. 연구 범위 및 내용

### 4.1 1차년도 (2027): 통신·분산 기반 강화

**세부 과제 1-1: Zenoh 기반 소방 현장 통신 표준화**
- WiFi 두절 환경에서의 메시 네트워크 통신 검증
- 기존 DDS 대비 소방 현장 적용성 비교 벤치마킹 (DDS vs Zenoh, 패킷 손실 시나리오)
- 소방 현장 통신 표준 프로파일 제안

**세부 과제 1-2: CBBA 확장 — 번들 임무 할당**
- 현재 CBBA 단일 임무 할당 → 복합 임무(드론 정찰 + UGV 진입) 번들 지원
- 이종 로봇 능력(capability) 기반 자동 매칭 알고리즘

### 4.2 2차년도 (2028): 지능화·자율화

**세부 과제 2-1: MARL(다중 에이전트 강화학습) 통합**
- 시나리오: 단일 화재 → 복합 장애물 → 다중 화재(MARL 3단계 커리큘럼)
- 참조: Cyborg/Griffith 연구팀 MARL 99.67% 자율 성공률 재현
- Isaac Lab 환경에서 정책 학습 → Gazebo 이전 검증

**세부 과제 2-2: 이종 플랫폼 자동 온보딩**
- 신규 로봇 플랫폼 추가 시 PlatformInterface 자동 탐색·등록
- Open-RMF 통합 검토 (이종 로봇 플리트 표준)

### 4.3 3차년도 (2029): 실증 및 표준화

**세부 과제 3-1: 119리빙랩 실증**
- 국립소방연구원 119리빙랩 환경 적용 (이지향 팀장 협력)
- TurtleBot4 2대 + PX4 드론 2대 이종 편대 실증
- 소방관 조작성 평가 (UX 테스트)

**세부 과제 3-2: 표준 문서 및 오픈소스 공개**
- 소방 로봇 ROS2 인터페이스 표준 제안 (소방청 검토)
- ARGOS 플랫폼 v5.0 공개 + 개발자 가이드 배포

| 지표 | 목표값 | 근거 |

| 임무 할당 응답 시간 | 1초 이하 | CBBA 알고리즘 |
| 화재 탐지 정확도 | mAP50 0.95 이상 | AI-Hub 데이터셋 기반 |

### 5.2 정책·산업적 효과

- **HR-셰르파 100대 보급(2027~2030) 오케스트레이션 소프트웨어 공급**:
  현대로템 하드웨어 + ARGOS 소프트웨어 조합으로 국산 소방 로봇 생태계 구축
- **오픈소스 공개**: 국내 소방 로봇 스타트업·연구소의 공통 기반 기술 제공
- **소방관 부담 경감**: 로봇 1대당 1인 조작 → 1인이 편대 전체 지휘

### 5.3 현장 소방관 관점

- 진입 불가 구역(고온, 연기, 붕괴 위험) 자율 정찰 → 인명 피해 감소
- 실시간 웹 모니터링 UI (현재 daegufire.ai.kr/argos)로 지휘관 상황 인식 향상
