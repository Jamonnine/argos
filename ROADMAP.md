## 프로젝트 비전

**이종 군집 소방 로봇 오케스트레이션 플랫폼** — 드론, UGV, 보행형 로봇이 한 팀으로 화재 현장을 자율 탐색하고, 화점을 감지하며, 실시간으로 정보를 공유하는 Supervised Autonomy 시스템.

### 핵심 차별점
- **이종(Heterogeneous)**: 드론+UGV+보행형 조합 — 각 플랫폼의 강점 결합
- **오케스트레이션**: "무엇을" 결정하는 지휘관 + "어떻게"를 자율 수행하는 로봇
- **현장 기반 설계**: 현직 소방관이 실제 소방 지휘체계를 소프트웨어로 매핑
- **오픈소스**: Apache 2.0 — 공공기관·수상·상업 사용 가능

```
1. 현재 Phase 확인 → 아래 "개발 Phase" 테이블에서 현재 위치 파악
2. 해당 Phase의 다음 미완료 세션(S-XX) 확인
3. 세션 시작: WSL 기동 → colcon build → 해당 세션 목표 실행
4. 세션 종료: 이 파일 + argos-tutor.md 진도 갱신 → git commit → wsl --shutdown
```

**현재 위치**: `Phase A~E 전체 완료 → Phase G(고도화) + 외부 제출 병행`
**실적**: 813테스트 + mAP50=0.718 + Nav2 SUCCEEDED + Phase D/D+/E 전체완료 + 기술서·논문 초안

## 병렬 개발 프로토콜

> 세션당 코드 생성 7.5배 가속 확인 (2026-03-19 첫 적용)

```
Step 1: 병렬 코드 생성 (3~5분)
  - 에이전트 3개를 worktree 격리로 동시 실행
  - 사전 파일 분담으로 충돌 방지
  - 각 에이전트: 코드 작성 + 구문 검증

Step 2: 메인 통합 + 빌드 (10~15분)
  - py_compile 전체 검증
  - colcon build --symlink-install

Step 3: 순차 테스트 + 디버깅 (30~60분)
  - Gazebo 기동 → 검증 스크립트 순차 실행
  - 실패 시 SendMessage로 해당 에이전트 재개하여 수정
```

**규칙**:
- 에이전트별 수정 파일을 사전 배정 (동일 파일 교차 수정 금지)
- launch 파일, config 파일, 노드 코드는 각각 다른 에이전트에 배정
- 메인이 최종 빌드/Gazebo 테스트 담당 (서브에이전트는 WSL2 접근 불가)

### Phase D+ 참조
- **HR-셰르파 공식 제원**: `knowledge/firefighter/unmanned-fire-robot.md` (NFRI 발표 1차 자료)
- **HR-셰르파 호스 제약 리서치**: `knowledge/projects/research/2026-03-20-hr-sherpa-hose-constraints.md`
- **소방호스**: 65mm 충수 시 ~56kg/본, 꺾임 시 유량 10%↓/압력 15%↓, 완전 접힘=유로 차단

### 호스 제약 5대 규칙 (오케스트레이터 반영)
1. **Hose-Aware Path Planning**: 급커브 금지 (최소 회전반경 제한)
2. **진입 깊이 제한**: 호스 잔여 길이 = 최대 진입 반경 (100m)
3. **역할 분리**: 호스공급 전담 UGV + 진압 UGV + 호스 없는 드론(정찰)
4. **호스 충돌 회피**: 다른 로봇이 호스를 밟거나 넘지 않게 장애물 취급
5. **충수 후진 금지**: 물 찬 상태에서 후진 = 호스 꼬임 → 전진 경로만 할당
